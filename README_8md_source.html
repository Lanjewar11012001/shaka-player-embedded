1 # Shaka Player Embedded
2 
3 Shaka Player Embedded is a framework that runs [Shaka Player][] in native (C++)
4 apps. This gives a cross-platform interface to Shaka Player allowing native
5 apps to use the player. Your native apps can now be built with the same
6 features, behavior, and API as your Shaka-based web apps.
7 
8 In addition to the C++ API, we provide a high-level wrapper for iOS called
9 ShakaPlayerView which can be used from either Objective-C or Swift.
10 
11 See our [documentation][] and [iOS tutorial][].
12 
13 [Shaka Player]: https://www.github.com/google/shaka-player
14 [documentation]: https://google.github.io/shaka-player-embedded/
15 [iOS tutorial]: https://google.github.io/shaka-player-embedded/md_shaka_docs_tutorial_setup.html

18 ## Platform support
19 
20 We only support iOS at this time, but other platforms could be added.
21 
22 Because many on our team use Linux, there is a Linux port included to make it
23 easier to work on non-iOS-specific features. Linux is not, however, a
24 first-class target platform at this time.

55 ## Setting up for development
56 
57 1. The source is managed by Git at
58 <https://www.github.com/google/shaka-player-embedded>. You will need these
59 tools available on your development system:
60 
61 * Git v1.7.5+
62 * Python v2.7
63  * Requires installing package `enum34`
64 * autoconf 2.57+
65  * m4
66  * automake 1.7+
67  * perl
68 
69 This does not require a C++ compiler as it uses a pre-compiled version of
70 clang downloaded from Google Cloud Storage. However, it does require a C++11
71 compatible C++ library (installed with most compilers, e.g. gcc).
72 
73 2. Install Chromium depot tools, which contains ninja and other required tools.
74 See <https://www.chromium.org/developers/how-tos/install-depot-tools> for
75 details.
76 
77 3. Get the source
78 
79 ```shell
80 git clone https://github.com/google/shaka-player-embedded.git
81 cd shaka-player-embedded
82 ```
83 
84 We use git submodules to manage third-party dependencies. You can run
85 `git submodule update` (or pass `--recurse-submodules` to `git clone`) to
86 download all the submodules; or you can wait and have the `configure` script
87 download only those you need.
88 
89 4. Configure and Build
90 
91 You need to run `configure` to setup configuration settings. You can pass
92 --help to see some of the options you can use. We don't support in-tree builds;
93 you need to run `configure` in another directory.
94 
95 Once `configure` is done, you can build using `build.py`. There is a Makefile
96 that will run the script for you if you prefer.
97 
98 ```shell
99 mkdir foo
100 cd foo
101 ../configure
102 make
103 ```
104 
105 5. Copy the shared library
106 
107 Once it is built, you can use the resulting shared library.
108 
109 On iOS, there will be a `ShakaPlayerEmbedded.framework` folder that you use;
110 there is also a `ShakaPlayerEmbedded.FFmpeg.framework` bundle that you need to
111 include in your app, but you shouldn't use it directly since we don't maintain
112 ABI for it.
113 
114 On other platforms, there is a `libshaka-player-embedded.dylib` or a
115 `libshaka-player-embedded.so` file you can use. Like iOS, there is a
116 `libshaka-player-embedded.ffmpeg.*` file you need to include too.
117 
118 If you are using a custom `--eme-impl`, you need to copy the respective
119 shared libraries too, if needed.

122 ## Running the checks
123 
124 If you intend to send a pull-request, you need to ensure the code complies with
125 style guidelines. This is done with the `shaka/tools/presubmit.py` script,
126 which can be run with `make check`.
127 
128 To run all the checks, you also have to have `clang-tidy` installed. It needs
129 to be installed on `PATH`, or you can pass `--clang-tidy` to the script to tell
130 it where to find the binary.
